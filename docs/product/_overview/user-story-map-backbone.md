# Coffee AI Mentor - User Story Map Backbone

<!--
  ・ユーザー：アプリ利用者・運営者・開発者などのステークホルダーを指す。
  ・ユーザーストーリーマップ：大きな価値・目的までの流れを俯瞰しながら、小さな価値を並べ、優先順位を決めて価値の最大化と一貫性を狙う
    ・シナリオ→アクティビティ→ステップ→ストーリーの順に考える。
      ・シナリオ：ユーザーがあるゴールを達成するまでの流れ
        ・分量：20～30字程度
        ・Happy Path（正常な成功シナリオ）を優先
        ・寄り道（追加・削除・変更・キャンセル・設定・エラー）は基本的に関連するステップのストーリーや受け入れ条件で扱い、
          そのためにアプリを立ち上げるレベルならアクティビティとして独立させる

      ・アクティビティ：シナリオを構成する行動のまとまり
        ・分量：20～30字程度
        ・個数：1シナリオあたり3～7個程度

      ・ステップ：アクティビティを構成する具体的な行動
        ・分量：20～30字程度
        ・個数：1アクティビティあたり3～7個程度

      ・ストーリー：ステップに関連する実装・検証可能な価値・能力（ユーザー・ビジネス・品質の価値。数日で実装が完了する量）
        ・分量：40～60字程度（As a・I want・so that／Job to be done）
        ・個数：1ステップあたり3〜7個程度
        ・ストーリー例：
          ・重要なエラー（リカバリ・エラーメッセージ表示など）
          ・重要な分岐（外部連携・ビジネスルール・管理系（ロール・権限・CRUD・設定）など）
          ・全体共通の非機能要件
          ・バックグラウンド処理（バッチ・ジョブなど）
          ・アクセシビリティ系
          ・計測系（KPI・A/Bテストなど）

      ・受け入れ条件：ストーリーの価値・能力に関する条件・ルール・具体例
        ・個数：1ストーリーあたり3〜7個程度（ガーキン記法／箇条書き）
        ・受け入れ条件例：
          ・単純なエラー（境界値テスト・バリデーションエラー・エラーメッセージ表示など）
          ・ストーリー内に収まる分岐（操作対象・入力制限・一時エラーのリトライなど）
          ・ストーリー内に収まる考慮（UIフォーカス・マスキングなど）
          ・ストーリー固有の非機能要件
 -->

## Scenario 1：利用者がアプリの存在を知ってから、継続をジャッジするまで

### Activity 1-1：自分ごととして興味を持つ

- コーヒー関連の情報を見ている中で、このプロダクトの紹介に出会う。
- タイトルやキャッチコピーをざっと読み、自分のコーヒーの悩みと関係あるか確かめる。

### Activity 1-2：自分の課題とプロダクトを照らし合わせる

- 説明ページ全体を一通り眺めて、情報量と雰囲気をつかむ。
- 利用条件や費用、必要な時間感などを確認し、自分の生活リズムと照らし合わせて考える。
- 現在のやり方や他のサービスと比べて、このプロダクトを試す価値を判断する。

### Activity 1-3：初回セットアップを最小ストレスで完了する

- 利用登録に進む。
- ユーザー名・経験レベルなど必要最低限の情報だけ登録する。
- 初回利用のための簡単な説明やガイドに、必要な範囲だけ目を通す。

### Activity 1-4：ガイド付きの「最初の1杯」を記録し、AIメンターとの体験を一度味わう

- 最初の1杯の記録を始める。
- ガイドに沿って、その一杯の基本情報（豆・抽出方法など）を入力する。
- その一杯の飲んだ印象を、シンプルな評価や短い言葉で記録する。
- AIとの対話に入り、味の特徴や抽出方法との関連を整理していく。
- AIによって整理された内容を踏まえて記録しなおす。
- AIによる次の一歩の提案を一通り読む。

### Activity 1-5：数回の利用を通じて、負荷と楽しさ・有用性のバランスを見極める

- 忙しさや気分に合わせて、時間や内容の深さを調整しながら使ってみる。
- 豆や抽出方法なども変えて試してみる。
- 数日分の記録がたまったタイミングで、一覧を見返す。
- 気になった記録は加筆修正や削除する。
- 数日分の記録内容やAIのフィードバックを眺めながら、手間や気づきを見極める。
- より快適かつ効率的な利用方法や利用頻度などを試す。

### Activity 1-6：初期体験を振り返り、続けるかどうか判断材料を整理する

- これまでの記録の中から、印象的だった数件を選び、内容とフィードバックを落ち着いて読み返す。
- 良かった点と続けにくかった点を、頭の中または簡単なメモで整理する。
- このプロダクトを使う前と比べて、「味の捉え方」や「淹れ方の意識」がどう変わったかを言語化してみる。
- ここまでの体験を踏まえて、「続ける」「頻度を下げる」「一旦やめる」など、今後の付き合い方を一度決める。

### Activity 1-7：継続する場合、日常ルーティンへの組み込み方を決める

- 「コーヒータイムのどの瞬間にこのツールを使うか」を具体的に決める。
- 決めたタイミングを忘れにくくするための仕掛け（リマインドの方法やトリガー）を用意する。
- 「最低限ここだけはやる」という自分ルール（例：平日の朝だけ記録／1日1杯だけ詳しく書く）を決める。
- 数日〜1週間ほど、そのルールに沿って実際に使い続けてみて、続けやすさを体感で確かめる。
- 体感に合わせて、ルールやリマインドの強さ・頻度を微調整し、自分にとって無理のない形に整える。

## Scenario 2：利用者が一杯を学習サイクルとして回す

### Activity 2-1：一杯を「ちゃんと味わって記録する」と決めてアプリを開く

- その日のコーヒータイムが来たら、「今日は記録しながら飲む」と意図的に決める。
- 記録を取るタイミングを、その日の流れに合わせて決める（抽出前／抽出中／飲んだ直後など）。
- 決めたタイミングになったら、コーヒーの記録を始める行動に移る。

### Activity 2-2：味単位で記録を振り返り、自分の気分に合った淹れ方を振り返る

- 直近の記録をいくつかまとめて振り返る。
- その日の気分や体調を思い浮かべ、それに近い状況で飲んだ記録を探す。
- 見つけた記録の内容（抽出条件・味の印象）を読み、今日の一杯の参考として頭に入れる。

### Activity 2-3：今日のカップの条件をざっくり整える（豆・レシピ・気分をセットする）

- 今日使う豆を、手元の候補の中から1つ選ぶ。
- 過去の記録や自分の好みをもとに、豆量・挽き目・時間などをざっくり決めてレシピの方針を固める。
- 今日の目的や気分を一言で言語化し、そのイメージに合う淹れ方の方針を決める。

### Activity 2-4：コーヒーを淹れて飲みながら、第一印象をシンプルに記録する

- 決めたレシピに沿ってコーヒーを淹れる。
- 一口目から数口にかけて、味・香り・口当たりを意識しながら味わう。
- その一杯のおいしさや軽さ／重さなどを、シンプルな指標でひとまとまりに評価する。
- その場で浮かんだキーワードや短い感想を、忘れないうちに書き留める。

### Activity 2-5：AIメンターと対話しながら、味の要素と言葉を整理する

- 味の第一印象を記録したあと、「もう少し整理したい」と思ったタイミングでAIとの対話を始める。
- AIからの質問に答えながら、香り・酸・甘み・ボディ・後味などの要素ごとに印象を分解していく。
- AIが提案する表現や比喩の中から、自分にしっくりくるものを選び、自分の語彙として取り込む。
- 整理された言葉や指標を、そのカップの正式な記録としてまとめて残す。

### Activity 2-6：AIと抽出条件と味の関係を整理し、原因仮説を立てる

- AIの指摘や自分の感覚をもとに、「この味に効いていそうな要因」をいくつか候補として挙げる。
- 候補の中から「次回検証したい原因」を1つに絞り込む。
- 絞り込んだ原因を、「今回立てた仮説」として記録に紐づけておく。

### Activity 2-7：次の一杯で試す小さな実験を決めて保存する

- 立てた仮説に合わせて、「次回どのパラメータをどの程度変えるか」を具体的な一歩に落とし込む。
- 決めた変更内容を、次の一杯のための「実験メモ」としてまとめて残す。
- 次回のコーヒータイムでその実験を思い出せるよう、必要なリマインドやメモの仕組みを用意する。

### Activity 2-8：週・月単位で記録を振り返り、自分の好み・上達パターンを確認する

- 週末や月末などの区切りで、「ここまでの記録をまとめて振り返る」時間を確保する。
- 一定期間の記録を一覧で俯瞰し、評価や語彙、レシピのパターンをざっくり眺める。
- AIがまとめた振り返りコメントや傾向分析を読み、自分の体感と照らし合わせる。
- 振り返りで気づいたポイントを、一言でもよいので「これから意識したいこと」として書き留める。
- その気づきに沿って、次の週・月の「実験テーマ」や使い方の方針を1つ決める。
